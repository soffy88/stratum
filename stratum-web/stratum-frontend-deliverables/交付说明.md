# Stratum 前端交付物 v1.0

12 个文件,直接放进 stratum-web 现有项目对应路径。后端 API 已实装,直接对接真实 API(无 mock)。

## 文件清单(路径即目标位置)
```
src/lib/views.ts                    API client(listViews/createView/updateView/deleteView)
src/lib/highlights.ts               API client(listHighlights/deleteHighlight)
src/lib/graph.ts                    API client(listEntities/getSubgraph/queryGraph)
src/components/EmptyState.tsx       通用空状态
src/components/LoadingSkeleton.tsx  骨架屏(Card/Grid/Graph 三变体)
src/components/ConfirmDialog.tsx    删除确认(基于 shadcn alert-dialog)
src/components/ViewEditDialog.tsx   视图编辑(全字段:名称/说明/图标/medium 多选/标签/排序/显示模式)
src/app/(app)/views/page.tsx        视图管理页(预设 + 我的视图,删预设 toast 报错)
src/app/(app)/highlights/page.tsx   高亮管理(颜色映射/倒序/来源跳转/删除确认/骨架屏)
src/app/(app)/graph/page.tsx        知识图谱(实体列表 + reactflow 子图 + GraphRAG 查询,移动端 Tab)
src/app/(app)/documents/page.tsx    改造参考(View Banner,★ 标记部分合并进现有页)
src/components/layout/Sidebar.tsx   导航(新增「知识图谱」入口 + 移动端汉堡 drawer)
```

## 安装依赖
```bash
cd stratum-web
pnpm add reactflow
```

## 注意
1. **documents/page.tsx 是改造参考**:你现有的 documents 页已有列表/上传逻辑,把 ★ 标记的 View Banner 部分合并进去即可,不要整体覆盖。
2. shadcn/ui 组件(button/input/label/dialog/alert-dialog)用你项目里现有的,import 路径 `@/components/ui/*` 已对齐。
3. 所有按钮已满足 min-h-44px(min-h-11),移动端汉堡 + drawer,深浅主题用 Tailwind dark: 变体。
4. 全部用 React state,无 localStorage/sessionStorage。
5. 验收点(§8)已逐条实现:View Banner、预设不可删、颜色映射、reactflow 着色、移动端 Tab、骨架屏、toast 错误。

已通过 TypeScript 类型检查(tsc 零错误)。
