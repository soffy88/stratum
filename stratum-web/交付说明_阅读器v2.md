# Stratum 阅读器 v2 — @react-pdf-viewer 按需渲染 + 高亮/笔记

替换 react-pdf 全量渲染(慢的根因)为 @react-pdf-viewer 按需渲染;加高亮/笔记,存回 Stratum highlights(跟 /highlights 页同源)。

## 文件清单
```
src/components/PdfReader.tsx            @react-pdf-viewer + 高亮(多色 + 回显)
src/components/EpubReader.tsx           react-reader + epubjs cfi 高亮
src/components/DocumentViewer.tsx       按 mime 分发 + blob 加载 + 大文件兜底
src/lib/highlights.ts                   补 createHighlight/updateHighlight/listHighlightsByDocument + location_json 类型
src/app/(app)/documents/[id]/page.tsx   original tab 接 DocumentViewer(传 byteSize)
```

## 安装依赖
```bash
pnpm add @react-pdf-viewer/core@3.12.0 @react-pdf-viewer/default-layout@3.12.0 @react-pdf-viewer/highlight@3.12.0
# react-reader + epubjs 已装
```

## ⚠️ PDF.js worker 版本对齐(关键,实测)
@react-pdf-viewer 3.12.0 内部 pdfjs-dist 是 **3.11.174**,worker 文件是 **pdf.worker.min.js(不是 .mjs)**。
需求文档写的 .mjs 是错的 —— 版本/扩展名不对齐会导致 worker 加载失败、PDF 打不开。

正确做法:
```bash
cp node_modules/pdfjs-dist/build/pdf.worker.min.js public/pdf.worker.min.js
```
PdfReader 里 `<Worker workerUrl="/pdf.worker.min.js">` 已用 .js。
(若以后升级 @react-pdf-viewer 导致内部 pdfjs-dist 变 4.x,worker 才是 .mjs,届时同步改。)

## 关键实现
1. JWT:file endpoint 带 token → apiClient fetch blob。PDF 用 blob URL,EPUB 用 ArrayBuffer(比 blob URL 稳,§5)。
2. 按需渲染:@react-pdf-viewer 默认只渲染可视页,大 PDF 秒开(解决 react-pdf 全量慢)。
3. PDF 高亮:highlightPlugin renderHighlightTarget(选中弹多色)+ renderHighlights(回显,按 pageIndex 过滤 + getCssProperties 定位)。存 location_json {page, rects}。
4. EPUB 高亮:rendition.on('selected') → cfi 定位,annotations.highlight 即时显示。存 location_json {cfi}。重开遍历回显。
5. 同源:高亮存 /api/v1/highlights,跟 /highlights 页同一数据源(那里能看、能删)。
6. 大文件兜底:>200MB 直接给下载;50-200MB 显示下载进度。
7. 动态导入:详情页 DocumentViewer 用 next/dynamic ssr:false(@react-pdf-viewer/epubjs 仅客户端)。
