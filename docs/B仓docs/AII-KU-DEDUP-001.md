# AII KU 去重机制设计：A 仓 → B 仓

> **Doc ID:** AII-KU-DEDUP-001
> **定位:** B 仓的核心工序——A 仓的 KU（按书、有重复、中文）如何去重整合，灌入 B 仓（跨书、无重复、英文）。这是 B 仓"自然生长"的入口机制，也是 concept canonical/超边/本性的数据准备。
> **依据决策（已定）:**
> - 流程：**翻译英文 → KU 判同（先行）→ 同点内容整合（相同丢/不同补/标出处）→ 存 B 仓 KU → 概念归一 → 谱社区 KC**。
> - **KU 判同先行**（用 KU 名+内容判，不强依赖概念已归一）；去重后再做概念归一——两层分开，同一套"真正同一"判据。
> - 内容整合 **宁冗余不误删**（不同侧面必须保留=越读越厚；只有真相同才丢）。
> **状态:** 设计供审，先不改代码（等 B 仓建库后实现）。
> **命门:** **误删 = 丢知识**（把不同侧面当相同删，丢了"越读越厚"）。和 concept canonical"宁碎片不错合"同理——**宁冗余不误删。**

---

## 一、为什么这是 B 仓的核心工序

- B 仓是"自然生长"的知识有机体：KU/KC 不是一次抽好的，是随 A 仓喂书不断长出来的。
- **去重机制 = B 仓生长的入口**：每本书的 KU 进来，和已有的整合（相同丢/不同补），B 仓越来越厚。
- 质量门级重要：去重错了（误删/错并），B 仓的概念归一、超边、本性全建在错误数据上。**去重是 B 仓干净的第一道关。**

---

## 二、完整流程（6 步）

```
A 仓 KU（按书、有重复、中文为主）
  │
  ├─ 步骤1 翻译英文（B 仓语言统一）
  │     中文 KU → 英文；本就英文的保留
  │     ★副作用（好）：很多"名不同"变"同名"
  │        机会成本 → Opportunity Cost（= 另一本的 Opportunity Cost）
  │     ★忠实翻译（不改内容）+ 术语准（经济/数学标准英文术语）
  │
  ├─ 步骤2 KU 判同（先行，真正同一判据，作用在 KU 名+内容）
  │     判"两个 KU 是不是讲同一个点"：
  │       · KU 名语义同（英文，翻译后；Opportunity Cost = Opportunity Cost）
  │       · KU 内容讲同一个点（不只名同，内容也是同一知识点）
  │     ★用 AII-CONCEPT-IDENTITY-001 的"真正同一"思路（语义+本质，非字符串）
  │     ★存疑 → 判为不同（不强合）——宁冗余不误判
  │
  ├─ 步骤3 同点 KU 内容整合（★核心难点：相同丢/不同补/标出处）
  │     判为同点的多个 KU → 内容整合：
  │       · 相同内容部分 → 丢弃（不重复存）
  │       · 不同内容部分 → 补充（越读越厚）
  │       · 标出处（sources：这部分来自哪本书）
  │     ★宁冗余不误删（见 §三）
  │
  ├─ 步骤4 存为 B 仓 KU（英文、无重复、内容多书整合）
  │     refined_ku：英文 natural_text + sources（多出处+各贡献什么）
  │
  ├─ 步骤5 概念归一（在去重 KU 上做 concept canonical）
  │     KU 已去重 → 在干净 KU 上归一概念（AII-CONCEPT-IDENTITY-001 真正同一判据）
  │     ★KU 判同（步骤2）和概念归一（步骤5）两层分开，同一套判据
  │
  └─ 步骤6 谱社区 KC（按概念聚类）
        去重 KU + 归一概念 → 谱社区（主题 KC，跨书）
  │
  ▼
B 仓 KU/KC 自然生长（新书来 → 重复 1-6 → 越来越厚/全）
```

## 三、★步骤3 核心难点：内容整合"判相同不误删"

> 这是整个去重最难的，也是命门所在。

### 3.1 难点：判"哪部分相同（丢）哪部分不同（补）"
- **误删（判太松）**：数分 ε-δ 严格定义 vs 同济应用导向定义，**都是"导数定义"KU，但内容不同**（一讲严格性、一讲应用）。若判"相同"删一个 → 丢了"越读越厚"。
- **漏删（判太严）**：两书讲同一点，表述不同实质相同 → 没丢，B 仓有冗余。

### 3.2 ★宁冗余不误删（命门，权衡）
- **误删 = 不可逆丢知识**：删了的内容（不同侧面）找不回，且 B 仓"越读越厚"的价值受损。
- **冗余 = 可逆**：重复内容以后能再合（升级判定后重跑/人工）。
- → **误删 >> 冗余 的危害**。判内容相同**宁严勿松**：拿不准是否相同 → 当"不同"保留（补充），不当"相同"删。
- 和 concept canonical"宁碎片不错合"、超边"宁缺毋附会"同一命门家族——**AII 全局：宁可冗余/碎片/漏，不可误删/错合/附会。**

### 3.3 内容整合的判定（可操作）
```
两个同点 KU 的内容 → 切成内容片段/要点 → 逐片段判：
  · 片段语义相同（讲同一个事实/定义/例子）→ 相同 → 保留一份（标多出处）
  · 片段语义不同（一个讲严格性、一个讲应用）→ 不同 → 都保留（各标出处）
  · ★拿不准 → 当"不同"保留（宁冗余不误删）
判定用 LLM（给两片段，判"是不是讲同一个事"），但：
  · LLM 判"相同"才丢——门槛高（确信讲同一个事才丢）
  · LLM 判不准/拿不准 → 保留（不丢）
```
- **本质维度区分（呼应判同）**：ε-δ 严格定义 和 应用定义，是导数定义的**不同侧面/深度**（本质不同的讲法）→ 保留。不是"换个说法的同一内容"。

### 3.4 整合产物（refined_ku.sources 记录）
```
导数定义 KU（B 仓，整合后）:
  natural_text: [严格定义部分] + [应用导向部分]（不同内容都保留）
  sources: [
    { book: 数分华东师大, version, contributed: "ε-δ 严格定义" },
    { book: 同济高数,   version, contributed: "应用导向定义 + 几何意义" }
  ]
```
- 一个 KU，内容是多书不同侧面的并集，每部分标出处——**这就是"越读越厚"的数据落地。**

## 四、步骤1 翻译：简化器 + 注意点

### 4.1 翻译是判同的简化器
- 中文 KU 翻译成英文后，**很多"中英文差异导致的名不同"消除**：机会成本→Opportunity Cost，和另一本英文 KU 同名 → 判同难度大降。
- 所以"翻译先行"不只是 B 仓英文统一的要求，也**降低了 KU 判同/概念归一的难度**（统一语言后，名更可能一致）。

### 4.2 翻译注意点（命门：翻译不能失真）
- **忠实翻译**：翻译是"换语言"不是"重新表述"——不改变内容（不增不减不曲解）。
- **术语准**：经济/数学术语有标准英文（弹性=elasticity, 导数=derivative, 机会成本=opportunity cost）——术语翻错 → 判同/归一全错。
- **可用 gemma/本地模型翻译**（0 成本），但术语校验（术语词典对照）。
- ★翻译失真 → 后面判同/整合/归一都错 → 翻译质量要验证（抽查翻译忠实否、术语准否）。

## 五、两层判同的关系（KU 判同 vs 概念归一）

| | KU 判同（步骤2） | 概念归一（步骤5） |
|---|---|---|
| 判什么 | 两个 KU 是不是讲同一个点 | 两个概念是不是同一概念 |
| 依据 | KU 名 + KU 内容 | 概念名 + 判别维度 + 上下位 |
| 时机 | 先行（去重前） | 去重后（在干净 KU 上） |
| 判据 | 真正同一（AII-CONCEPT-IDENTITY-001） | 真正同一（同一套） |
| 破循环 | KU 用自己名+内容判，不强依赖概念已归一 | 在去重 KU 上做，数据已干净 |

- **同一套"真正同一"判据，两个层面（KU 层、概念层），分时做**。
- KU 判同先行破了"去重依赖归一、归一依赖去重"的循环——KU 判同有自己的依据（KU 名+内容），不必等概念归一。

## 六、自然生长（B 仓的本质）

```
新书 → A 仓抽取（原始 KU）
     → 翻译英文 → KU 判同（和 B 仓已有 KU 比）
        · 是已有点 → 内容整合（相同丢/不同补，B 仓该 KU 变厚）
        · 新点 → 新增 B 仓 KU
     → 概念归一（新概念归一/已有概念补别名）
     → 谱社区 KC 更新（新 KU 入簇/簇调整）
B 仓 KU 越来越厚（多书不同侧面累积）、KC 越来越全（主题随书扩）
```
- A 仓固定（一本书抽完就定），B 仓生长（跨书整合，永远在长）。
- 这正是"死知识 → 有机体"：不是堆 N 本书的 KU，是融成一个越来越厚的知识体。

---

## 七、命门

1. **宁冗余不误删**（核心）：内容整合判"相同"宁严勿松；拿不准当"不同"保留。误删丢知识（不可逆），冗余可逆。
2. **翻译不失真**：忠实翻译 + 术语准；翻译错则后面全错，要验证。
3. **KU 判同先行破循环**：用 KU 名+内容判，不强依赖概念归一；两层同一套判据分时做。
4. **真正同一才整合**：KU 判同/内容整合/概念归一，都用"真正同一"（语义+本质，非字符串）；存疑不合并/不删。
5. **自然生长**：B 仓 KU/KC 随书长；A 仓固定 B 仓生长，各归本质。

---

> **一句话**：A 仓 KU 进 B 仓 = 翻译英文 → KU 判同先行（名+内容，真正同一判据）→ 同点内容整合（相同丢/不同补/标出处，**宁冗余不误删**）→ 存 B 仓 KU → 概念归一 → 谱社区 KC。两层判同（KU 层、概念层）同一套判据分时做，破互依赖循环。B 仓 KU/KC 自然生长——多书不同侧面融成越来越厚的知识体。**误删丢知识是命门，宁冗余不误删。**
