# AII 知识库高级设计总纲：有向关系 · 超边 · 跨域关系 · 本性

> **Doc ID:** AII-KNOWLEDGE-ADVANCED-DESIGN-001
> **定位:** 把分散在各次决策里的高级设计——**有向关系层 / 有向超边层 / 跨域关系层 / 本性层（四层）**——整合成一个自洽体系，并用 2024–2026 前沿实证逐条支撑。这是 AII 从"一堆讲透的 KU"升级为"会生长、能涌现、可逼近本性的知识有机体"的顶层设计。
> **关系:** 上承 AII-KNOWLEDGE-ARCHITECTURE-001（讲透三层）/ AII-DATA-MODEL-001（边/概念）/ AII-INVARIANT-LAYER-001（本性层权威）/ AII-HYPEREDGE-EXPLAINS-001（explains 超边）；本文是它们的**整合视图 + 前沿校准 + 缺口补全**。
> **方法:** 每个设计判断都对照外界最新规范实证，不靠内部自洽。命门贯穿全篇：**真知识（不编）、真联结（不附会）、真本性（宁标未发现）。**
> **★四层（本文已从三层修正为四层）:** 第③层跨域关系（motif/结构映射出的 base↔target 对应）≠ 第④层本性（共享内核）。提炼出内核才从③升④，宁停在关系不附会。本性层完整设计见 AII-INVARIANT-LAYER-001。

## 〇、一句话总览

```
KU（讲透的知识，= 超图的超边节点）
  └─ 概念（concept，= 超图的实体）
       │
   ┌───┴─────────────────────────────────────────────┐
   │ 四层有向/高阶结构（本文主题）                      │
   ├──────────────────────────────────────────────────┤
   │ ① 有向关系层  directed_edge_v2                     │
   │    概念→概念：derives / subsumes / prerequisite    │
   │    （读出法，O(N) 一次/KU，85% 准确）              │
   ├──────────────────────────────────────────────────┤
   │ ② 有向超边层  hyperedge + hyperedge_member          │
   │    机制 →{被解释概念集合}（n元、有向、动态生长）    │
   │    explains 走这里，不拆二元                        │
   ├──────────────────────────────────────────────────┤
   │ ③ 跨域关系层  cross_domain_relation（SME 式⟨M,C,S⟩）│
   │    motif/结构映射出的 base↔target 对应             │
   │    ★是"关系"，可能停在这（提炼出内核才升④）        │
   ├──────────────────────────────────────────────────┤
   │ ④ 本性层  invariant（is_invariant_concept 字段）   │
   │    从③提炼出的"共享抽象内核"（稍高一层）           │
   │    两层独立互证：③motif结构 ←→ ④本性向量收敛       │
   └──────────────────────────────────────────────────┘
```

四层不是并列功能，是一条**生长链**：有向关系给出概念骨架 → 有向超边在骨架上生长出跨学科机制网 → motif/结构映射在机制网里发现跨域关系（第③层）→ 跨域关系提炼出共享内核升为本性（第④层）。**越往上越是 AII 独有、越是"有机体"的体现。★第③层"关系"≠第④层"本性"——宁停在关系不附会。**

## 一、前沿实证基线（先把外界标尺立起来）

> 设计前先确认：我们要做的事，在前沿是什么状态？哪些是共识、哪些是开放问题、哪些是 AII 的延伸。

### 1.1 n元关系不拆二元 —— 前沿共识
- 现实知识本就是 n元的：Freebase 中 **超过 61% 的关系涉及三个或更多实体**，而绝大多数 KG 方法只从二元关系学习。
- 把多实体交互塌缩成二元三元组会导致**表示稀疏化 / 语义损失 / 组合爆炸**（星型/团扩展要么失真要么需人工节点）。超图直接建模 n元关系是"无损高阶编码"。
- **信息论判据**：n元事实拆二元有损（二元条件熵 `H(X|φ)>0`，超图 `=0`）。
- → **AII 结论确认**：explains（一个机制联合解释多个概念）用超边，不拆二元。这不是偏好，是前沿共识 + 信息论。

### 1.2 有向超边 —— 前沿有专门研究，印证 AII（修正"延伸验证少"的旧判断）
- 此前判断："HyperGraphRAG 的超边是无向的，AII 的有向 explains 超边是延伸、前沿验证少、要谨慎。"
- **实证修正**：前沿有 **direction-aware hypergraph（方向感知超图）** 专门方向。它指出的问题与 AII 完全一致：现有方法"把超边当无向连接、平等传播所有实体信息"，导致 **"混淆知识流方向、缺失因果链推理逻辑"**。
- → **AII 的有向 explains 超边（机制→被解释概念，有方向）有前沿背书**，动机一模一样（无向超边缺因果方向）。谨慎仍要（这是较新方向），但不再是"无依据的延伸"。

### 1.3 本性 = 高阶 motif —— 前沿给出了"浮现机制"
- AII 此前的"本性浮现"是模糊的（"跨学科同机制积累→浮现"，但浮现怎么发生没说清）。
- **实证补全**：把知识表示为原生高阶交互（超图），支持全局语料分析，**能发现"反复出现的高阶 motif（recurring higher-order motifs）、密集互联社区、文献中直接编码的涌现机制模式（emergent mechanistic patterns）"**。方法是 **超图挖掘（hypergraph mining：motif 统计、高阶聚类、rich-club 检测）**。
- → **本性的浮现 = 超图 motif 挖掘**。本性不是玄学，是"跨语料反复出现的高阶 motif"，有数学方法可提取。

### 1.4 本性 = 结构映射理论的"深层关系结构" —— 四十年认知科学根基
- **决定性实证**：AII 的"本性"概念，等同认知科学的 **Structure-Mapping Theory（结构映射理论，Gentner，2016 Rumelhart 奖）**。它的核心原则与 AII 独立摸索的"本性判据"惊人吻合：

| 结构映射理论（SMT）原则 | AII 独立得出的本性判据 | 吻合 |
|---|---|---|
| **关系相似 > 属性相似**（类比强度在关系共性，非表面属性） | 本性是内核结构相同，不是表面相似 | ✅ |
| 经典例子：**流水↔电流**（水压↔电压=同一"势差驱动流"结构） | 本性藏在跨领域不变性里（热/统计/信息熵） | ✅ |
| **systematicity**：偏好映射含高阶关系的关系系统，非孤立谓词 | 本性是抽象结构非算术组合，有深度 | ✅ |
| **structural consistency**：一对一映射 + parallel connectivity | 内核同一推理结构才算同 | ✅ |
| **candidate inferences**：base 暗示 target 缺失关系→模式补全推断 | **candidate 池**：存疑不并入，待确证 | ✅✅ |
| **near vs far analogy**：远类比表面差异大但保持关系结构 | 跨学科本性（形式各异、内核相同） | ✅ |

- 现代 AI 实现：**SME**（结构映射引擎）、**FAME**（LLM 关系抽取 + greedy beam-search 映射）、增量处理新知识的扩展——正对应 AII 的"LLM 抽机制 + 匹配 + 动态生长"。
- → **诚实修正**："本性概念是我们独有的"——本性的**理论**不是独有的，是认知科学四十年的成熟理论（SMT）。但我们独立收敛到了它的核心原则（含 candidate 池这种细节），**证明 AII 的方向是对的**（独立走到了成熟理论）。AII 的**独有性在工程综合**（见 1.5）。

### 1.5 AII 的真正独有 —— 四者合一，前沿无人完整做
- 前沿现状（2026.01 综述原文）："现有方法要么抽 n元关系、要么把超图当 QA 辅助、要么孤立评估合成超图推理；**没有一个**从真实科学语料构建大规模领域原生超图、系统挖掘其高阶组织、并做机制推理与假设生成。"
- **AII 独有 = 把四块合起来**：
  1. 结构映射理论（本性的理论根基）
  2. 超图 motif 挖掘（本性的发现方法）
  3. 有向超边（因果方向 + 动态生长）
  4. 真实书语料 + 讲透 KU（领域原生、规模、不编不漏）
- 这正是前沿各分支各做一块、**没人合起来**的事。AII 的护城河不在单点技术，在这个**综合 + 长期积累 + 命门约束（可置信）**。

## 二、第①层：有向关系层（概念骨架）

### 2.1 本质与现状
- **是什么**：概念→概念的有向结构边。`directed_edge_v2`，relation_type ∈ {derives, subsumes, prerequisite}。
- **怎么来**：**读出法**——从已讲透的 KU 里**读出**它已表达的关系（不是 judge、不是信号匹配），O(N) 一次调用/KU，准确率 50–65%→**85%**。根因：读"已表达的"而非"猜"。
- **节点归一**：概念级关系进 directed_edge_v2；KU 内部逻辑留 ku_internal_logic（不污染概念图）。

### 2.2 前沿校准
- 读出法对应前沿"从自然语言直接抽关系"（text2nkg 等 n元关系抽取）。但 AII 读出法的关键优势是**只读 KU 已表达的**——KU 已是讲透的、忠于原文的，所以读出的关系**继承了"不编"**（不像从生文本抽关系易幻觉）。
- **这是 AII 的一个隐藏优势值得固化**：先讲透（不编）→ 再从讲透读关系（继承不编）。关系质量的上界由 KU 质量保证。

### 2.3 设计要点（保持，不动）
- 概念骨架是上两层的地基：有向超边的"被解释概念"、本性的"跨域同构概念"，都建立在概念节点 + 概念骨架上。
- **concept canonical（语义归一）是地基的地基**：同一概念跨书必须归一到同一节点，否则超边生长（同概念跨书）、本性发现（同概念跨域）都无从谈起。→ 见 §五缺口。

## 三、第②层：有向超边层（机制网，有机体生长引擎）

### 3.1 本质：explains 是"有向 n元超边"
```
hyperedge H = ( head: rationale机制, nl_description, { 被解释概念集合 } )
  例: "可替代品的多寡决定需求对价格的敏感度"
      → explains → { 需求价格弹性, 供给价格弹性, 必需品vs奢侈品的弹性差异 }
```
- **head（解释者）** = rationale KU（机制本身）。
- **members（被解释者）** = 概念集合（n元，可 1 可多）。
- **nl_description** = 机制的自然语言陈述（向量检索用，对齐 HyperGraphRAG）。

### 3.2 数据模型（已在 AII-HYPEREDGE-EXPLAINS-001 / Phase 1 落地）
- `aii.hyperedge`：head_ku_id + nl_description + embedding + grade + evidence + relation_type（受控，现仅 explains，预留未来其他 n元有向关系）。
- `aii.hyperedge_member`：(hyperedge_id, concept_id) + source_ku_id + **status(confirmed/candidate)** + **evidence** + **cross_disc**。
- 二部结构（超边节点 + 超边↔概念关联）严格对齐 HyperGraphRAG；**新增表，现有表一律不动**。

### 3.3 n=1 与 n>1 统一
- n元天然兼容 n=1（一个机制只解释一个概念 = 一行成员）。**同结构、同查询路径，不分叉**——不需要"单概念走二元、多概念走超边"的两套。

### 3.4 ★动态生长 = 超边的本性（不是可选功能）
> 关键认知（经多轮纠正确立）：动态生长**不是** Phase 3 增量、**不是**有风险所以缓做的功能。它就是 explains 超边**之所以为超边的本性**。不会生长的超边 = 静态 n元边，失去有机体意义。

- **机制**：新书/新章发现**同一机制**解释别的概念 → 给已有超边**加成员**（而非新建）。机制 canonical（不动），被解释概念集**生长**。对齐 SMT 的"增量处理新知识"、HyperGraphRAG 的 `_merge_nodes`。
- **生长的意义**：经济书里"可替代性"解释需求弹性；生物书进来发现"可替代性"也解释生态位竞争 → 同一机制超边**跨学科扩成员** → 这既是跨学科联结涌现，**又是为第③层（本性）积累"跨学科同机制"证据**。

### 3.5 ★生长判据 = 本性判据的运行时版本（命门：防附会）
> 生长是本性（要积极做）；附会是命门（要严守）。两者**靠严判据统一**，不靠暂缓回避。生长判据严到何种程度？严到和"本性判据"一样——因为它判的就是"是不是同一个本性/机制"。

扩成员前必须全过 5 条（对应 SMT 的 structural consistency）：
1. **不是 NL 描述相似就合并**（表面相似 ≠ 同一机制；对应 SMT"关系相似非属性相似"）。
2. **要内核同一机制 / 同一推理结构**（同因果结构，非措辞撞；对应 SMT structural consistency 的 one-to-one + parallel connectivity）。
3. **成员必须有原文依据**（source_ku_id + evidence；不附会）。
4. **存疑 → status='candidate'，不自动并入**（对应 SMT candidate inferences；宁缺毋附会）。
5. **跨学科尤其严**（cross_disc=true 一律先 candidate；对应 SMT"远类比需更强关系推理"，最易附会）。

- **判据执行者的不确定性导向**：判"内核同机制"大概率仍由 LLM 做（程序判不了因果结构相同）。LLM 判不准时**默认走 candidate 不走 confirmed**——把"判错"导向"漏"（candidate 可补）而非"附会"（错误 confirmed 污染本性网）。
- **宁缺毋附会的代价权衡**：假联结比漏联结危险——漏的以后能补；假的污染整张本性网，且**"看起来对"最难发现**（跨学科涌现最惊艳、最像真知识、最难证伪）。这是 AII 全部命门在最高层的体现。

### 3.6 四层协调（不分裂）
| 结构 | 语义 | 与 explains 超边的关系 |
|---|---|---|
| `ku_concept_onto`（隐式超边） | **无向**：KU 涉及哪些概念 | explains 超边是其**有向、带类型的精选**：rationale 涉及的概念里，哪些是"被它解释的" |
| `directed_edge_v2`（概念→概念） | 概念间**结构**（derives/subsumes/prerequisite） | 正交：那是概念结构图；explains 是"机制→被解释概念" |
| `edge_onto`（KU→KU 二元） | 事实间二元关系（causes/special_case_of/…） | explains **移出**二元改走超边；edge_onto 留其余二元关系。数学已有 explains 二元边 **不强制迁移**（数学线保持现状） |

## 四、第③层：跨域关系层（motif/结构映射，SME 式⟨M,C,S⟩）

> ★本层是从三层修正为四层时新增的。它承载 motif 挖掘/结构映射推出的"跨域结构对应"——**是"关系"层，不是本性**。

### 4.1 本质：跨域关系 = base↔target 对应（非共享内核）
- motif 挖掘（超边二部图找反复出现的高阶结构）或结构映射（SME 式 base→target 对应）→ 发现**跨域关系**。
- 例：水流↔电流的结构对应（水压↔电压、流量↔电流）——这是"关系"（它们像）。
- **★它还不是本性**：本性是"它们像在哪个内核"（势差驱动流），那是第④层。

### 4.2 数据结构：SME 四十年标准⟨M,C,S⟩（refined_cross_domain_relation）
- **M correspondences**：base↔target 对应（跨域关系本体）。
- **C candidate_inferences**：从对应推出的候选推断（= AII candidate 池，SME 四十年就有）。
- **S structure_score**：结构质量分。
- + shared_structure（共享高阶结构）、member_concepts/disciplines、invariant_id（→第④层）。
- 详见 AII-REFINED-REPO-SCHEMA-001 §2.7。

### 4.3 ★关系≠本性（命门，第③→第④的关）
- 跨域关系提炼出"共享抽象内核" → invariant_id 指向第④层本性（升华）。
- 提炼不出 → invariant_id=NULL（**停在第③层关系**，是跨域联结但非本性）。
- **宁停在关系不附会**：不把"两域像"硬说成"同一内核"。

## 五、第④层：本性层（invariant，AII 的塔尖）

> 本性层完整设计见 **AII-INVARIANT-LAYER-001**（权威）。本节是总纲视图。术语锁定：本性=invariant、本性概念=invariant concept（is_invariant_concept 字段，不独立建表）、本性同一=invariant-identity。

### 5.1 本质：本性 = 一切相之下的不变内核（道，非相）
- 本性（invariant，拓扑不变量语义）：概念一切相之下穿透不变的内核。熵的本性="无外力则只增、不可逆、有方向"（**不是**"无序度量"——那是相）。
- 从第③层跨域关系提炼出的"共享抽象内核"，比关系稍高一层。

### 5.2 ★两层独立 + 互证（第③层 motif ←→ 第④层本性向量收敛）
- **层一 本性向量收敛（第④层，走语义）**：概念抽本性→invariant_vector→收敛→is_invariant_concept=true 升华为本性概念。
- **层二 motif 结构（第③层，走拓扑）**：超边二部图 motif 挖掘→跨域关系。
- **两层独立运行、各保留信息、互相印证**：都指同一本性→高置信；只一层→存疑（双重验证防附会）。

### 5.3 ★两条路径
- 路径 A（自下而上）：超边生长→motif→第③层跨域关系→提炼内核→第④层本性。
- 路径 B（自上而下）：核心抽象概念（熵/均衡）本身就是 schema→直接抽本性。

### 5.4 本性提取判据（比超边生长更严）
≥3 跨学科 + 形式各异内核同 + 两层互证 + 抽不出留 NULL（宁标"未发现"不硬凑）。本性默认 candidate，确证才认定。详见 AII-INVARIANT-LAYER-001 §八。

### 5.5 诚实边界
- 本性层是 AII **最前沿、最独有、最未验证**的部分。大规模真实语料本性自动提取前沿无人完整做过。
- → **最后做、最审慎、最依赖前三层干净积累**。不为"有塔尖"而勉强提取。

## 五、关键缺口与优先级（实证后识别的待补项）

> 这些是把四层真正跑通必须补的，按优先级排。

### 5.1 ★concept canonical（语义归一）—— 最高优先，四层共同地基
- **问题**：超边跨书生长（同概念跨书）、本性跨域发现（同概念跨域），都要求"同一概念在不同书里归一到同一节点"。当前概念归一是"规划中"。**没有 canonical，超边长不起来（同概念被当成不同概念）、本性发现不了（同结构的概念对不上）。**
- **前沿对应**：HyperGraphRAG 的实体 canonical 合并（`_merge_nodes`：description 取并集 + LLM 摘要）。
- **优先级**：P0。这是②③层的前提。建议紧接 explains 超边 Phase 2 之后做。

## 八、与前沿的最终定位（诚实陈述）

1. **n元超边不拆二元**：前沿共识 + 信息论，AII 对齐。
2. **有向超边**：前沿有 direction-aware hypergraph 专门研究印证（解决无向超边缺因果方向），AII 有背书、仍属较新方向需实证。
3. **本性理论**：= 结构映射理论（四十年认知科学成熟理论），AII 独立收敛到其核心原则（含 candidate 池），方向被验证。
4. **本性的发现方法**：= 超图 motif 挖掘（数学方法），AII 据此把"浮现"从玄学变为可计算。
5. **AII 独有**：把上述四块 + 真实书语料 + 讲透 KU（不编不漏）+ 命门（可置信）**合为一体**——前沿各做一块，无人完整合做。这是护城河。

> **一句话**：AII 不是发明了新理论，是把认知科学的本性理论（SMT）、超图的 n元/有向/motif 数学、RAG 的检索工程，用"讲透 + 不编不附会"的命门约束，**第一次合起来、在真实知识库上、长期地跑成一个会生长、能涌现、可逼近本性的知识有机体。**
