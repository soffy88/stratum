# AII 概念判同升级设计：真正的同一才合一

> **Doc ID:** AII-CONCEPT-IDENTITY-001
> **定位:** 升级 M0（concept canonical）的**判同逻辑本身**——从"字符串/向量相似 + LLM 判"升级为"**真正同一才合一**"的本体级同一性判定。这不是给判别词硬闸打补丁，是把判同设计提升到"概念本质是否相同"的高度。
> **触发:** M0 全局归一破坏性测试暴露真错合——price-inelastic ← income-inelastic（price/income 混）、unitary-income ← unitary-price、increasing opportunity cost 并进基础机会成本（子概念过合）。根因：单靠 LLM 判同不稳（LLM 倾向"强行匹配"），判别词黑名单是补丁（挡已知词，挡不住没想到的）。
> **状态:** 设计供审，先不改代码。审过 + 全量前定对，再动手。
> **命门:** 错合 = 地基级污染（上层超边/本性全错）。**宁留碎片不错合。真正同一才合一。**

## 一、前沿实证：错合是头号问题，解法正是"本体约束"

> 升级前先确认：概念判同/实体对齐的前沿，怎么避免错合？

### 1.1 错合（false merge）是实体对齐的头号问题，且一半是"类冲突"
- OntoEA（顶会）实证：标准 benchmark 上，**错误预测的映射里 42.2%–55.7% 是"类冲突（class conflicted）"**——把不同类的实体合了。
- → **AII 的 price-inelastic vs income-inelastic 正是"类冲突"**（不同弹性类型被合）。这不是我们的个例，是实体对齐的普遍头号问题。

### 1.2 前沿解法 = 本体约束（类层级 + 类互斥），正是"真正同一才合"的方向
- **OntoEA**：联合嵌入 KG + 本体，**利用"类层级（class hierarchy）和类互斥（class disjointness）避免错误映射**"。
- **PARIS**（经典）：通过 **"functionality（功能性）" + 比较属性值** 计算等价概率——**关键属性值不同 → 等价概率低 → 不合并**。
- → 你提的"判别维度对齐 + 非上下位"在前沿有成熟对应：**判别维度 ≈ PARIS 的属性值比较；非上下位/互斥 ≈ OntoEA 的 class hierarchy + disjointness。**

### 1.3 ★LLM 判同会"强行匹配"——印证 AII 的撞坑
- 2025 论文明确：**"LLM 倾向于强行做出选择，即使没有正确匹配存在"**。
- → 正是 AII 撞上的：llama-3.3-70b 把 price-inelastic 和 income-inelastic 判了 SAME（强行匹配）。**这是 LLM 已知缺陷，不是个例。**
- 前沿做法：LLM 做"过滤/验证候选"，但**要有本体约束（disjointness）兜底**——和 AII"LLM 判 + 程序闸兜底"一致，但要把"程序闸"从黑名单升级为本体约束。

### 1.4 结论
"真正同一才合一" = 在向量粗筛 + LLM 判语义之外，**加本体级约束**：判别维度对齐（PARIS）+ 类层级/互斥（OntoEA）。前沿验证，且和 AII 已有体系（判别维度、有向关系层、结构映射）打通。

## 二、判同的根本问题：相似 ≠ 同一

### 2.1 现状判同（相似度判断）的缺陷
```
现状：向量相似（同discipline 余弦≥0.85）→ LLM 判同 → 合并
缺陷：
  ① 向量相似 = 表面/语义近，不等于"同一概念"
     （price-inelastic 和 income-inelastic 向量很近——都在讲"无弹性"）
  ② LLM 判同会强行匹配（前沿确认）
  ③ 判别词黑名单是补丁（挡 price/income，挡不住没列的判别维度）
```

### 2.2 ★"真正同一"的定义（正向，不靠堵漏）
两个概念 A、B **真正同一**，当且仅当全部满足：
```
① 判别维度全对齐：A、B 在每个关键判别维度上取值相同
     （不是名字像，是"对什么的弹性/什么的成本/什么时期"逐维度相同）
② 非上下位：A 不是 B 的子类/特化，B 也不是 A 的（子类≠同一）
③ 非互斥：A、B 不属于互斥的并列类（价格弹性 ⊥ 收入弹性）
④ 核心结构相同：关系结构同（对应结构映射，不只表面）
```
- 缺任一 → 不同概念 → 不合并（宁留碎片）。
- 这是**正向定义同一**（满足全部才合），不是反向堵漏（黑名单挡已知）。

## 三、升级后的判同流水线（四道关，逐层收紧）

```
候选对 (A, B)  ← 向量粗筛（同discipline 余弦≥阈值）得到候选
   │
   ├─ 关1 判别维度对齐（程序，确定性）── PARIS 式属性值比较
   │     抽 A/B 的判别维度取值 → 任一维度取值不同 → DIFFERENT（不进后续）
   │     ★挡 price/income/supply/demand/短长期/arc-point... 一切判别维度
   │     ★正向：比较维度取值，非黑名单匹配词
   │
   ├─ 关2 类层级/互斥（用第①层有向关系 directed_edge_v2）── OntoEA 式
   │     A subsumes B 或 derives 关系 → 上下位 → 不合并（是父子非同一）
   │     A、B 在互斥并列类（同父下的不同子）→ 互斥 → 不合并
   │     ★挡 increasing opportunity cost 并进基础机会成本
   │
   ├─ 关3 LLM 判同语义（前两关过了才问 LLM）── 但收紧
   │     给 A/B 名 + 定义 + 判别维度 → 判"是不是同一概念"
   │     ★反义/方向反/不确定 → DIFFERENT（前沿：LLM 倾向强行匹配，故默认 DIFFERENT）
   │     ★LLM 只在"判别维度已对齐 + 非上下位"的窄候选里判，错判空间已被前两关压小
   │
   └─ 关4 高风险 → candidate（不自动合）── 宁缺毋附会
         跨学科 / LLM 置信不足 / 任一关存疑 → status='candidate'，人工/更多证据再确认
         确信（四关全过 + 高置信）→ confirmed，真合并
```

**逐层收紧的逻辑**：程序关（1、2）确定性挡掉大部分错合（判别维度、上下位）→ LLM 关（3）只在窄候选里判语义（错判空间已压小）→ 高风险关（4）兜底进 candidate。**LLM 不再是唯一闸，是"被前两关保护的窄判断"。**

## 四、关1 详解：判别维度对齐（升级判别词硬闸）

### 4.1 从"黑名单词"到"判别维度取值"
- **旧（补丁）**：黑名单 [price, income, supply...]，名字含不同黑名单词 → DIFFERENT。缺陷：只挡列进去的。
- **新（升级）**：抽概念的**判别维度 + 取值**，逐维度比较：

```
判别维度（discriminative dimension）= 区分同族概念的关键属性
  弹性族：  弹性对象（price/income/cross）、测量法（arc/point）、时期（short/long-run）
  成本族：  成本对象（production/investment）、增量性（basic/increasing/marginal）
  ...（按概念族，可由 LLM 抽取或维度词典）

判定：A、B 同族 → 逐判别维度比取值 → 任一维度取值不同 → DIFFERENT
  price-inelastic：{弹性对象: price}  vs  income-inelastic：{弹性对象: income}
    → 弹性对象 price≠income → DIFFERENT ✓（确定性挡住，不靠黑名单）
  Price elasticity arc：{对象:price, 测量法:arc} vs Price elasticity point：{对象:price, 测量法:point}
    → 测量法 arc≠point... 但 arc/point 是同一概念的测量变体（见 4.2 特判）
```

### 4.2 ★测量变体 vs 判别维度（细节，防过严）
- 有些"维度不同"是**测量/表述变体**（arc/point 弹性、本质同一概念的两种算法），该合。
- 有些"维度不同"是**本质不同**（price/income 弹性，不同概念），不该合。
- **区分**：判别维度分两类——
  - **本质维度**（弹性对象、成本对象）：取值不同 = 不同概念 → DIFFERENT。
  - **表述维度**（测量法 arc/point、单复数、大小写）：取值不同但本质同 → 可合。
- 维度词典标注每个维度是"本质"还是"表述"。**本质维度不同→挡；表述维度不同→放行进 LLM。**

### 4.3 实现
- 判别维度抽取：LLM（给概念名+定义，抽其判别维度取值）或维度词典（经济/数学常见维度）。**0 成本本地可做。**
- 关1 是程序确定性比较（抽取后比取值），不依赖 LLM 判断的稳定性。

## 五、关2 详解：类层级/互斥（复用第①层有向关系）

### 5.1 用 directed_edge_v2 判上下位
- 第①层已有概念→概念的 derives/subsumes/prerequisite。
- **A subsumes B（A 含 B）或 A derives B → A、B 是上下位，不是同一 → 不合并。**
  - increasing opportunity cost（递增机会成本）derives/特化自 机会成本 → 不合并（是子概念）。
- 这把"过合子概念"用已有的有向关系确定性挡住。

### 5.2 类互斥（disjointness）
- 同父类下的不同子类 = 互斥并列（价格弹性、收入弹性、交叉弹性都是"弹性"的子类，彼此互斥）。
- **互斥并列 → 不合并。**
- 互斥关系可由判别维度推出（同族但本质维度取值不同 = 互斥），与关1 呼应。

## 六、命门：宁留碎片不错合（地基级）

### 6.1 为什么错合比碎片危险（再强调）
- **错合 = 地基污染**：两个不同概念合成一个 → 上层所有超边（被解释概念错）、本性（同构概念对错）全错，且**难发现**（合了就看不出原来是两个）。
- **碎片 = 暂时没并**：以后能再并（升级判同后重跑、或人工）。可恢复。
- → **错合不可逆的污染 > 碎片可恢复的不全**。判同宁严勿松。

### 6.2 不确定一律不合（导向碎片，不导向错合）
- 四关任一存疑 → 不合并（或 candidate），绝不自动合。
- LLM 判不准 → 默认 DIFFERENT（前沿：LLM 倾向强行匹配，必须默认不合兜底）。
- **把"判错"导向"漏（碎片，可补）"，不导向"错合（污染，难恢复）"。**

### 6.3 dry_run 强制
- 全局归一一律先 dry_run（算 + 打印"会合并哪些"，不落库）。
- 人工/抽查 dry_run 结果无错合 → 才真合。
- **杜绝"破坏性测试落库错合"重演**（M0 已踩，2 组落库需回滚）。

## 七、与 AII 体系的统一（不是孤立的判同模块）

| 升级判同用到的 | AII 已有 | 说明 |
|---|---|---|
| 判别维度 | concept level/discipline + 维度抽取 | 关1，PARIS 式属性值比较 |
| 类层级/互斥 | **directed_edge_v2（第①层有向关系）** | 关2，OntoEA 式，复用有向关系判上下位 |
| 核心结构相同 | 结构映射理论（本性同款判据） | 关3，关系结构同非表面 |
| candidate 池 | 超边生长的 candidate 机制 | 关4，宁缺毋附会，与超边一脉 |
| 向量粗筛 | concept.vector（BGE-M3） | 候选生成 |

> **判同不是独立模块，是 AII 体系的有机部分**：用第①层有向关系判上下位、用结构映射判结构同、用 candidate 池兜底——和超边、本性同源同命门。**地基的判同，和上层的联结/本性，守同一套"真正同一/真正同构才连"的原则。**
