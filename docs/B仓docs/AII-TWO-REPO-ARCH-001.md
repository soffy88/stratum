# AII 双仓架构设计：原始仓 / 精炼仓

> **Doc ID:** AII-TWO-REPO-ARCH-001
> **定位:** 确立 AII 的**两个互相独立、不会污染的数据仓库**——原始仓（raw，有重复 KU）与精炼仓（refined，无重复 KU，知识有机体本身）。这是数据架构的地基决策，影响 concept canonical / M1 超边 / M2 生长 / M3 本性**在哪个仓执行**。
> **触发:** M0 全局归一在原始仓（有重复 KU）上跑——位置不对。canonical/超边/本性应在去重后的精炼仓上跑，否则被原始重复污染。
> **决策（已定）:**
> - 精炼仓 = **独立数据库**（另一个 PG 库，最彻底物理隔离）。
> - KU 去重粒度 = **内容片段级**（相同内容丢/不同补/标出处）——机制细节另文（本文先定架构骨架）。
> - 当前原始仓全库 dry-run **停掉**（位置不对）。
> **状态:** 架构骨架设计供审。片段去重判定（核心难点）另文详。

## 一、为什么要两个仓（不能一个库混着）

### 1.1 一个库混着的污染
```
若原始 KU（有重复）和归一/超边/本性在同一库：
  ① canonical 污染：同概念有 5 份重复 KU → 归一要处理 5 份噪声
  ② 超边污染：explains 超边连到哪份重复 KU？歧义
  ③ 本性污染：motif 挖掘在"有重复 KU"的图上跑 → 高阶 motif 统计失真
     （重复 KU 制造虚假的"反复出现"，把噪声当 motif）
  ④ 向量污染：重复 KU 的向量挤在一起，影响相似度判断
```

### 1.2 两仓独立的好处
- **原始仓**：忠实保留每本书抽出的原始 KU（有重复、有噪声），是**原始证据/可追溯源**。不去重、不归一——它的职责是"忠实记录抽取了什么"。
- **精炼仓**：去重后的干净 KU，是**知识有机体本身**。concept canonical、超边、本性都建在这——在干净、无重复的图上跑，不被原始重复干扰。
- **互不污染**：精炼仓的归一/超边/本性，永远在干净数据上；原始仓的重复/噪声，永远进不了精炼仓的图。

## 二、两仓的职责与内容

### 2.1 原始仓（raw repository）
```
独立数据库（如 aii_raw）
内容：
  - 书源文件（或路径索引）
  - 每本书抽取的原始 KU（含重复——同概念多本书各抽各的）
  - 原始概念名（裸概念，未归一）
  - 抽取元数据（来自哪本书、哪章、抽取时间）
职责：忠实记录"抽取了什么"，可追溯。不去重、不归一。
向量化：原始仓自己的（如有需要）。
```

### 2.2 精炼仓（refined repository）
```
独立数据库（如 aii_refined）
内容：
  - 去重后的 KU（无重复；同内容只一份，标多出处）
  - 按主题 KC（谱社区）组织
  - 归一后的 canonical 概念（concept canonical 在这跑）
  - 超边 hyperedge（M1/M2 在这跑）
  - 本性 invariant（M3 在这跑）
职责：知识有机体本身。canonical/超边/本性全在这执行。
向量化：精炼仓独立的向量空间（干净 KU 的向量）。
```

### 2.3 两种 KC 在精炼仓的角色（呼应已有设计）
- **按章 KC**：书内固定结构（读这本书用），可保留在原始仓或精炼仓做"书视图"。
- **按主题 KC（谱社区）**：跨书主题，**是精炼仓的组织方式**——成员是去重后的 KU，跨书，标来源。**谱社区天然把"同主题 KU"聚到一起，对 concept canonical 有帮助**（提供"哪些 KU 讲同概念"的聚类线索）。

## 三、★数据流：原始仓 → 精炼仓

```
┌─────────────── 原始仓 (aii_raw) ───────────────┐
│  书源 → 抽取（程序WHAT + gemma WHY/HOW）         │
│       → 原始 KU（有重复）                        │
│       → 原始概念名（裸）                         │
└──────────────────────┬─────────────────────────┘
                       │
              ★去重合并（KU 内容片段级）
              · 相同内容 → 丢弃（不重复存）
              · 不同内容 → 补充进精炼仓的主题
              · 记录出处（这部分来自哪本书）
              ［判定细节：另文 AII-KU-DEDUP-001，核心难点见 §五］
                       │
                       ▼
┌────────────── 精炼仓 (aii_refined) ─────────────┐
│  无重复 KU，按主题 KC（谱社区）组织              │
│       → concept canonical（全局归一，在这跑）     │
│       → M1 explains 超边（在这跑）                │
│       → M2 超边生长（在这跑）                     │
│       → M3 本性 motif 挖掘（在这跑）              │
│       → 独立向量化                               │
└──────────────────────────────────────────────────┘
```

### 3.1 流的方向：单向 A→B（原始仓喂精炼仓）
- 原始仓是源，精炼仓是精炼产物。**单向流**：原始仓 → 去重 → 精炼仓。
- 精炼仓不回写原始仓（原始仓忠实保留原貌）。
- 追溯：精炼仓的每个 KU/内容片段，记录它来自原始仓的哪些 KU（哪些书）——**可追溯到原始证据**。

### 3.2 增量：新书进来
```
新书 → 原始仓（抽取，原始 KU 入原始仓，可能和已有重复）
     → 去重合并到精炼仓：
         · 新书的 KU 内容，和精炼仓已有同主题 KU 比
         · 相同部分丢弃、不同部分补充、标新书出处
     → 精炼仓的主题"越读越厚"（不同内容累积，相同不重复）
     → 触发精炼仓的 canonical/超边生长更新
```
- 这正是"越读越厚"（导数定义：数分 ε-δ 严格 + 同济应用导向**并存**，因为它们是"不同内容"补充进来，不是重复）。

## 五、片段级去重的核心难点（本文先标，另文详）

> 决策已定"内容片段级去重"，但**判定细节是独立难点**，另文 AII-KU-DEDUP-001 详。这里先标核心难点，避免遗漏。

### 5.1 核心难点：判"相同内容"既不误删又不漏删
- **判太松（相似当相同）→ 误删**：数分 ε-δ 严格定义 vs 同济应用导向定义，**都是"导数定义"但内容不同**（一严格一应用）。若判"相同"删一个 → 丢了"越读越厚"。
- **判太严（逐字才算相同）→ 漏删**：两本书讲同一点，表述不同实质相同 → 没去掉，还是重复。

### 5.2 ★和 concept canonical 异曲同工（同一类判同问题）
- concept canonical：判"两概念是不是真正同一"（节点级）。
- 片段去重：判"两段内容是不是真正相同"（内容级）。
- **都是"判同"问题，都怕"相似 ≠ 同一/相同"误判**。
- → 片段去重可复用 canonical 的"真正同一才合"思路（AII-CONCEPT-IDENTITY-001）：本质相同才判"相同内容"删，本质不同（严格 vs 应用）保留。**异曲同工，另文统一设计。**

### 5.3 谱社区对去重和 canonical 的双向帮助
- 谱社区按主题聚 KU → 把"可能重复的同主题 KU"聚到一起 → 帮去重找候选。
- 去重后的干净 KU → 帮 canonical（同概念 KU 不被重复干扰）。
- canonical 归一概念 → 帮谱社区确认 KU 同概念。
- **三者互相帮助，都在精炼仓内闭环。**

## 七、命门

1. **两仓不污染**：精炼仓的归一/超边/本性永远在干净（无重复）数据上；原始仓的重复/噪声进不了精炼仓的图。
2. **单向流 + 可追溯**：原始仓 → 精炼仓单向；精炼仓每个 KU 可追溯到原始仓证据（哪些书）。
3. **去重不误删**（核心命门，另文）：判"相同内容"宁严勿松——本质不同（严格 vs 应用）必须保留（越读越厚）；本质相同才删。**误删 = 丢知识；和 canonical 错合一样，宁缺毋误删。**
4. **canonical/超边/本性在精炼仓**：M0 位置修正，在去重后的干净仓做。

---

> **一句话**：AII 用两个独立数据库——**原始仓（忠实记录抽取，有重复）** 与 **精炼仓（知识有机体，无重复）**。数据单向 A→B（去重合并：相同丢/不同补/标出处），concept canonical 和 M1/M2/M3 全在精炼仓的干净数据上执行。两仓不污染：原始的重复噪声进不了精炼仓的图，精炼仓的归一/超边/本性永远干净。**地基的地基——先有干净的精炼仓，上面的归一、超边、本性才立得住。**
