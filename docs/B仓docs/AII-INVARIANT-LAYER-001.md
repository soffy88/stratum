# AII 本性层完整设计：invariant

> **Doc ID:** AII-INVARIANT-LAYER-001
> **定位:** 本性层（B 仓塔尖，**第④层**）的权威完整设计。第一次把本性层的**术语统一锁定** + **设计整合精准**。整合 AII-CONCEPT-NATURE-001（本性是道非相/本性≠本性概念/向量收敛）+ 后续厘清（关系≠本性、两条路径、两层独立互证、本性概念字段标记存储）。
> **四层定位:** ①有向关系 ②超边 ③跨域关系（motif/结构映射出的 base↔target 对应，SME 式⟨M,C,S⟩）④本性（invariant）。**本性是从第③层跨域关系提炼出的"共享抽象内核"，比关系稍高一层**，为方便区分单设第④层。第③层提炼出共享内核→升第④层本性；提炼不出→停在第③层关系（宁停在关系不附会）。
> **校正:** 三层总纲/实现（AII-KNOWLEDGE-ADVANCED-DESIGN-001 / -IMPL-001）此前把本性简化为"motif 挖掘直接得本性"——**偏差**（motif 出的是第③层跨域关系，非直接得本性）。本文校正为完整设计。
> **命门:** 本性是塔尖，错则污染最大。**关系≠本性（不附会）、抽不出留空（不硬凑）、宁标未发现。**

## 〇、★术语锁定（权威，不再混用）

> 本性层术语此前文档间混用（nature/invariant、本性同源/本性同一）。此处锁定，以本表为准。

| 中文 | 英文术语 | 锚定/语义 |
|---|---|---|
| **本性** | **invariant** | 拓扑学不变量语义——一切相之下的不变内核（**不是 nature**；nature=性质/特征=相） |
| **本性概念** | **invariant concept** | 多概念共有的本性升华而成（**不独立建表**，is_invariant_concept 字段标记） |
| **本性同一** | **invariant-identity** | 两概念共享同一本性的最深连接（identity=同一，**非 equivalence 等价/similarity 相似**；修正自旧称"本性同源/invariant-shared"） |

- **本性 = invariant，不是 nature**：nature 会被理解成"性质/特征"=相；invariant（拓扑不变量）才是"一切相之下的不变内核"。抽英文书更精确。
- 旧称 **本性同源 / invariant-shared 作废** → 本性同一 / invariant-identity。

## 一、本性是什么：道，非相

### 1.1 相 vs 本性（invariant）
- **相**：能被观测、描述、测量的样子（量是多少、公式长什么样）。**静态、可见。**
- **本性（invariant）**：驱动它、决定它命运的内在规律/必然趋势——它**必然怎么动、必然往哪去**。**动态、内在、决定性，穿透一切相都不变。**

**例（熵）：**
- 相：「对可能状态数/不确定性的度量」「S=k·lnW」——这些是熵的描述面向（相）。
- **本性（invariant）：「无外力下，只增、不可逆、有方向」**——熵真正的内核，不依赖你用哪种相描述它，换任何相这个"只增"都成立。第二定律、时间箭头系于此。

> **本性不是"它是什么样"（相），是"它必然怎么变"（道）。** 抽本性，抽的是概念的内在律动/必然趋势，不是定义。

### 1.2 拓扑不变量语义（为什么叫 invariant）
- 拓扑不变量：咖啡杯↔甜甜圈，形状（相）随意变形，亏格（洞数）不变——**不变量就是本性**。
- invariant 精确表达"一切相之下穿透不变的内核"，这是 nature（性质=相）表达不了的。

## 二、本性 ≠ 本性概念（关键区分）

| | 归属 | 地位 |
|---|---|---|
| **某抽象概念的本性（invariant）** | 绑在**单个**抽象概念上（熵的本性） | 是该概念的属性 |
| **本性概念（invariant concept）** | **多个**抽象概念**共有**的那个本性 | 凝结的独立实体 |

- **诞生条件**：只有发现「不同抽象概念共有同一本性」时，那个共有的本性才凝结成本性概念。
- **涌现，非预设**：积累到"出现共有"那一刻才产生。
- **判据同构**：多个具体概念的共性→抽象概念；多个抽象概念的本性的共性→本性概念。是否被下层多个成员**共有**。

## 三、★关系 ≠ 本性（防附会，最易错处）

> 这是本性层最易犯的错：把"跨域关系/相似"当成"本性"。

### 3.1 区分
- **跨域关系**：两个领域的结构对应/相似（水流↔电流）——**关系层面**（结构映射理论的 analogical mapping）。
- **本性（invariant）**：多领域共享的那个抽象内核本身（"势差驱动流"）——**本性层面**（结构映射理论的 schema abstraction，前沿原文："从映射中涌现的共同结构/不变量"）。

### 3.2 区别（讲透）
- 水流↔电流"像"= 发现了**跨域关系**（对应）。
- 它们"像在哪——都是势差驱动流"= **本性**（共享内核）。
- **关系是"这几个像"；本性是"它们像在哪个内核上"**——本性比关系深一层。

### 3.3 命门：宁停在关系不附会
- motif/结构映射推出的，**先承认是"跨域关系"**（领域对应）。
- **只有进一步提炼出"共享抽象内核"，才升为本性。**
- 提炼不出内核的，**停在"关系"层**（它仍有价值——跨域联结，但不是本性）。
- **把"关系"当"本性" = 附会**（把"两领域像"硬说成"同一内核"）。**宁停在关系，不把"像"当"同一内核"。**

## 四、★本性的两条发现路径

> 本性有两条独立路径（此前只设计了路径 A，漏了 B）。

### 4.1 路径 A：自下而上（聚类涌现）
```
超边动态生长 → 积累跨学科同机制实例（超边-概念二部图）
  → motif 挖掘（统计反复出现的高阶结构）
  → 跨域反复结构（落第③层 refined_cross_domain_relation，是"关系"层，§三）
  → 提炼共享内核 → 第④层本性（invariant）
```
- 适合：本性藏在分散实例里，需聚集才浮现。
- ★产物先是第③层"跨域关系/模式"，提炼出内核才升第④层本性（不直接=本性）。

### 4.2 路径 B：自上而下（核心抽象概念直推）
```
一个核心抽象概念（熵/均衡/势差驱动流/反馈）
  → 它本身就是高度抽象的 schema
  → 直接抽取它承载的本性（存概念的本性）
```
- 适合：本性"驻"在高度抽象的概念里。
- 前沿支撑：结构映射理论的 abstraction（"原子是中心力系统"——base/target 几无对象属性，概念本身即 schema）。**一个足够抽象的概念，本身就是本性，不必从实例聚类。**

### 4.3 两条路径都要过"关系≠本性"关
- 路径 A：motif 聚出的"反复结构"→ 提炼内核才是本性。
- 路径 B：从抽象概念推断 → 确认真是本性内核，不是泛泛关系。
- 殊途同归到本性，**都不把"关系/相似"直接当本性**。

## 五、★两层独立运行 + 互相印证

> 本性的发现有两个独立的层，各自运行、各保留信息、互相印证——不从属、非流水线。

### 5.1 两层
```
层一 本性向量收敛层（invariant，走语义，第④层）:
  概念 → 抽本性 → invariant + invariant_vector（带统一标记）
  → 多概念本性向量收敛（同向量=同本性）
  → 凝结/升华（is_invariant_concept=true）
  保留信息：本性、本性向量、本性概念

层二 motif 结构层 = 第③层跨域关系（走拓扑/统计）:
  超边生长 → 二部图 → motif 挖掘 / 结构映射
  → 跨域反复的高阶结构 / base↔target 对应（落 refined_cross_domain_relation，SME 式⟨M,C,S⟩）
  保留信息：跨域关系（对应/候选推断/结构分）（这本身是"关系"层有价值信息）
```

> **第③层（motif/跨域关系）和第④层（本性）是不同的层**：第③层是"base↔target 对应"（关系），第④层是"共享抽象内核"（本性）。第③层提炼出内核 → cross_domain_relation.invariant_id 指向第④层 invariant（升华）；提炼不出 → 留 NULL（停在第③层关系）。

### 5.2 互相印证（双向，非从属）
- **motif → 给本性发现提供数据**：motif 说"这几个领域结构反复出现"→ 信号：这里可能有共享本性 → 喂本性发现参考。
- **本性向量收敛 ← 被 motif 印证**：本性向量收敛说"这几概念本性同" + motif 说"这几领域结构反复" → **两独立机制都指同一处 → 高置信**。
- **只一层指向 → 存疑**（不轻易认定本性）。
- **谁也不从属谁**：本性向量收敛走语义、motif 走结构统计，各自独立成立。

### 5.3 为什么两层互证（命门：双重验证防附会）
- 单靠 motif（结构反复）：可能巧合结构相似（假模式）。
- 单靠本性向量收敛：可能向量误判（假收敛）。
- **两独立证据（一结构、一语义）都指同一本性 → 才高置信**。
- 这正是"宁标未发现不附会"：不靠单一机制认定本性，要两条独立路径互证。

## 六、数据结构（术语锁定 + 你定的字段标记）

```sql
-- 概念（B 仓 refined_concept 扩展）
concept:
  ...
  invariant_id   → 指向 invariant 表（该概念的本性，可空；抽不出留 NULL）

-- 本性（独立建表）
invariant:
  invariant_id
  statement              -- 本性是什么（如"无外力则只增、不可逆、有方向"）
  invariant_vector       -- ★本性向量，带统一标记（固定维度=本性，可被识别/收敛）
  is_invariant_concept   -- ★false=普通本性 / true=升华为本性概念（不独立建表！）
  member_concept_ids     -- 共有此本性的概念们（≥2 → 可升华 true）
  status                 -- candidate/confirmed（本性默认 candidate，确证才认定）
  created_at / updated_at
```

### 6.1 ★本性概念 = is_invariant_concept 字段（不独立建表）
- 本性概念**不另起表**，是 invariant 表里 **is_invariant_concept=true** 的那些本性。
- 普通本性 false，升华后 true → **同一条 invariant 记录，升华前后是同一条，只是标记变了**（不割裂、不冗余）。
- 本性概念直接显示：`WHERE is_invariant_concept=true`。

### 6.2 升华判据（false→true）
- 多个概念的 invariant_vector 收敛到同一条 invariant（同向量=同本性）。
- 且该 invariant 被 **≥2 个抽象概念共有**。
- → 该 invariant 标 is_invariant_concept=true（亮为本性概念）。

### 6.3 向量标记机制（本性同一自动成立的技术基础）
- 概念向量、KU 向量、本性向量各带可区分标记（本性用一个固定维度标识）。
- 同一本性，无论从哪个概念/路径/时间抽到，**向同一处收敛** → 共享即被发现，不靠 N² 逐对判断（避免造假）。

## 七、三种连接（绝不互相冒充）

| 连接 | 英文 | 是什么 | 强度 | 媒介 |
|---|---|---|---|---|
| 边 | edge | KU 间真实关系 | 局部强 | explains/causes |
| 概念共指 | — | 两 KU 含同一概念 | 弱（可发现性） | 同一概念 |
| **本性同一** | **invariant-identity** | 两概念共享同一本性，最深同构 | **最深** | 同一本性 |

- **本性同一（invariant-identity）的威力**：连接"名字完全不同、学科完全不搭"但本质同构的概念（经济"均衡"↔生态"稳态"）。
- 机制：本性作为独立维度，多概念指向同一本性 → 自动建立本性同一强联系（共享即连，不靠逐对判断）。
- **三种连接绝不互相冒充**：边/共指是"关系"，本性同一是"本性"——呼应 §三关系≠本性。

## 八、命门（本性层守则）

1. **关系 ≠ 本性**：motif/映射推出的先是关系，提炼出共享内核才升本性；宁停在关系不附会。
2. **抽不出留空**：invariant 抽不出留 NULL，不硬凑（硬凑=伪洞察="看起来对冒充真的对"）。
3. **本性概念涌现非预设**：≥2 概念共有才 is_invariant_concept=true，不预设本性概念。
4. **两层互证防附会**：本性向量收敛 + motif 结构，两独立证据都指同一本性才高置信；只一层存疑。
5. **本性默认 candidate**：本性是塔尖错则污染最大，默认 candidate，确证才认定。
6. **术语锁定**：invariant（非 nature）、invariant-identity（非本性同源）、本性概念用字段非独立表。

---

> **一句话**：本性 = invariant（拓扑不变量语义，道非相，熵的本性="只增不可逆有方向"非"无序度量"）。本性≠本性概念（多概念共有才升华，is_invariant_concept 字段标记，不独立建表）。**关系≠本性**（motif/映射推出先是跨域关系，提炼共享内核才升本性，宁停在关系不附会）。两条路径（概念直推/聚类涌现）、两层独立互证（本性向量收敛/motif 结构，都指同一本性→高置信）。本性同一（invariant-identity）是最深连接。**一切以真为底线——不让"看起来对"冒充"真的对"。**
