# AII 对 Stratum 交付 md 的章节结构质量规格

> **Doc ID:** AII-STRATUM-MD-SPEC-001
> **方向:** AII（下游，知识抽取） → Stratum（上游，源文件→md 转换）
> **依据:** AII 知识抽取需要"按章节"组织（章内建边、KU 打 chapter_id、跨章共现）。前置 = md 能被机器**可靠切出章节边界**。
> **现状(实证，见下"证据"):** Stratum 当前交付的 md **丢失/污染了章节结构**，AII 无法可靠分章 → 治本在 Stratum 转换质量，不在 AII 写脆弱的消歧分段器去猜。
> **接口:** `/home/soffy/shared/aii-to-stratum/md_quality_spec.json`（机器可读需求）+ 本文档（详细规格）。

---

## 1. 证据：当前 md 章节结构不可机器分段

实测样本：`Principles_of_Microeconomics_The_Way_We__01KVAJCX.md`（源为 PDF 教材，19 章，`source: stratum`）：

| 失败模式 | 实测 |
|---|---|
| **清晰章首标记只在目录** | `## **CHAPTER N**` 共 14 个，**全部在 0-1% 位置（目录区）**，正文无此标记 |
| **目录副本污染正文流** | 节标题 `## **9.1 Price Elasticity of Demand: An Overview**` 在目录块和正文**各有一份，都没有页码** → 无法用页码区分；朴素"首次出现"全部落进书首目录块（ch3-19 都解析到 0%） |
| **OCR 跑页眉当噪声** | 跑页页眉 `C H A P T E R  9 …`（字母间空格）重复 **340 次**，远多于 19 章 |
| **图表引用碰撞** | `Figure 9.1` / `FIGURE 9.1` / `As you saw in Figure 9.1` 含 "9.1"，与节号混淆 |
| **结果** | AII 用"章首/节号"任何朴素规则都切错（实测 19 章只识别出 9 章，且 ch9-17 落进一个 100 万字符的空洞）|

**根因:** PDF→md 转换是"接近纯文本 OCR"，丢了 PDF 本有的**结构信息**（书签/大纲树、标题字号层级、页码、正文 vs 目录区分）。

## 2. 要求：交付的 md 必须"章节结构可机器定位"

### R1 — 章/节用规范 markdown 标题层级（正文，非目录）
- 每章正文起点用**唯一、规范**的标题：`# Chapter N: <Title>`（H1，每书每章恰好一次，正文区）。
- 节用 `## N.M <Title>`（H2），子节 `### N.M.K`。
- **正文标题与目录条目必须可区分**（见 R2），不得用相同字面混入同一流。

### R2 — 目录(TOC)分离，不复制进正文流
- 目录**不得**把章/节标题行**逐字复制**进正文 markdown 流（当前最致命问题）。
- 二选一：① 转换时**剥除** TOC，正文里章/节标题只出现一次（正文那次）；或 ② 把 TOC 包进**显式标记**：`<!-- TOC START -->` … `<!-- TOC END -->`，AII 可整段忽略。

### R3 — 章节锚点机器可定位
- 每章起止可由**单一规范规则**定位（如"split on `^# Chapter \d+:`"即得 N 段，每段是一整章正文）。
- 不依赖"看后面是不是散文"这类脆弱启发式。

### R4 — OCR 书也要恢复结构（不只纯文本 OCR）
- 用 PDF 的**大纲/书签树**、**标题字号层级**、**页码**恢复章节层级，而非仅输出扁平 OCR 文本。
- **跑页页眉/页脚**（如重复的 `C H A P T E R N`、书名、页码）必须**识别并剔除**，不得当作标题或正文输出。

### R5（建议增强）— 章节清单 sidecar
- 在配对的 `.json`（或 md frontmatter）里附**章节清单**：`chapters: [{n, title, start_offset|start_line, sections:[{n_m, title}]}]`。
- 有了这个，AII **零猜测**直接按 offset 分章，最稳。

### R6 — 数学公式 LaTeX 保真
- 行内公式用 `$...$`、独立公式用 `$$...$$`，**公式内不被 OCR 拆散/乱码化**。
- 反例（当前坑）：`ε_d = (% Δq) / (% Δp)` 被 OCR 成 `e d 5 ... D q` 散字符 → 数学/经济 KU 质量崩。
- 上下标、希腊字母、分式、求和/积分须用 LaTeX 表达，不得退化为纯文本碎符。

### R7 — 图表规范占位
- 每个图/表用规范占位：`![Figure N.M: <caption>](<asset or omitted>)`，**caption 必须保留**。
- 反例（当前坑）：图被删后正文只剩 `the green rectangle` / `as shown above` → 产出**依赖图的死 KU**（不自足）。占位带 caption 后，AII 可把图引用降级、或据 caption 自足化。

### R8 — 表格结构保真（列对齐干净表，可按列 parse）
- 表格必须转成**列对齐的干净 markdown 表**：每行列数一致、表头单行、数据**可按行×列机器解析**；或明确占位 `![Table N.M: <caption>]`。
- **禁止**（实测踩到的烂表形态）：
  - **整张表挤进一个单元格**用 `<br>` 分行 —— 反例 `Table 2.1`：`|Food Preparation...14.42<br>11.55<br>10.55<br>8.23<br>...<br>Women only<br>24.65<br>...|`（整表体一格，不可按列拆）。
  - **列错位带空列** —— 反例 `Table 6A.1`：`|Food Units||Rental Housing (sq. ft.)|`、`|200||0|`、`|150||125|`（中列空、数据对不上表头）。
  - **表头重复跨列/被拆** —— 反例 `Table 6A.1`：表头 `**Table 6A.1 ...**` 跨 3 列重复，标题拆成 `|**Table **|**6A.1 **||`。
  - 散成残行 —— 反例 `Table 11.4 ... Taxable income is over |`（残片）。
- **验收标准**：① 每数据行列数 == 表头列数；② 表头单行(`|---|` 分隔紧跟其后)；③ 无整表挤一格的 `<br>` 堆叠；④ AII 能把表 parse 成 `rows×cols` 结构。
- **为何要列对齐**：涉及表格的 KU(命题)需用表的**结构化数据**完善（见 §6 后置增强）；烂表 parse 不动 → 数据吸收做不了。

### R9 — 剔除跑页眉/页脚
- 跑页页眉/页脚（重复书名、`C H A P T E R N`、页码）**必须识别并剔除**，不得作为标题或正文行输出。
- 反例（当前坑）：`C H A P T E R  9 …` 间隔字母重复 **340 次**，污染结构识别与抽取。

## 3. 验收标准（AII 侧可自动校验）
对每本交付 md，AII 用如下规则应能**确定性**切出正确章数：
1. `re.findall(r'^# Chapter (\d+):', md, re.M)` → 得到连续 1..N 且**每章恰好 1 次**（无重复、无缺号）。
2. 每章正文区不含其它章/节标题的**目录式重复**。
3. 切出的每章 char 区间合理（无某章吞掉其余全书的"空洞"）。
4. 跑页页眉计数 ≈ 0（不出现 `C␣H␣A␣P␣T␣E␣R` 这类间隔字母噪声）。
5. （R6）数学密集源里 LaTeX `$...$` 存在且非乱码（不出现大段散字符公式）。
6. （R7）图引用有 `![Figure ...]` 占位 + caption，正文无裸 `the green rectangle`/`as shown above` 悬空引用。
7. （R8）无表格残片行（`Table N.M ... |` 散尾、孤立 `|` 行）。
8. （R9）间隔字母页眉计数 ≈ 0。

满足 1-4（硬结构）即"章节结构合格"，AII 可上章节级摄取；5-8（R6-R9）为内容保真项。任一硬项不满足 → AII 标记 `chapter_structure_unreliable`，退回 Stratum 返工。

## 5. AII 侧自动质量门（闭环）
AII 在**摄取入口**（飞轮 `ingest_one`）对每本 md 跑 `md_quality_check`，按上述验收标准判定：
- **合格** → 进入抽取。
- **不合格** → **不抽**，自动写 rework 请求到 `aii-to-stratum/md_quality_spec.json` 的 `rework_request`（标不合格项 + 证据），等 Stratum 返工。
这是飞轮 690 本规模化的**入口质量门**：脏 md 挡在门外，不进抽取产生垃圾 KU。

## 6. 后置增强（待 Stratum 交付列对齐干净表后做，现在不做）

**增强需求：表数据完善 KU。** 实证表明正文与表的关系是「**正文命题（自足）→ 表格逐行数据（量化支撑）**」（如 *Table 15.1 shows diminishing marginal returns set in with the seventh doctor*）。当前涉及表格的 KU 处理**已对**（命题进 `content`、`Table N.M shows...` 进 `example` 降级，无逐行 dump、无碎片），但**只引用、不吸收表的实际数值** → 表的量化内容未进 KU。

增强方向（待干净表后）：把命题 KU 用对应表的**结构化数据 enrich** —— 或把表数据吸收进 KU、或作为 KU 的**结构化 evidence**（如实体-属性-值，可借鉴材料科学抽取 pipeline 的 entity-attribute-value，但适配 AII 的 KU/六分类体系）。

**硬前提：Stratum 交付 R8 列对齐干净表。** 当前烂表（挤一格/列错位）parse 不动 → **此增强现在做不了，标记为后置依赖**。

> **决策记录：** 当前表格处理（降级 example / 规律抽通则 / 无逐行 dump / 无碎片）**已正确，不动**；只在 Stratum 交付干净表后，再做"表数据吸收 KU"增强。

## 4. 优先级与范围
- **优先级:** high（章节级建边/chapter_id/跨章共现的硬前置；不解决则 AII 退回"50k 批 + 全书共现"，边界不贴章节，质量受限）。
- **范围:** 所有 `doc_type: book` 且有章节结构的源（教材/专著优先）。文章/论文（无章节）不适用。
- **不阻塞既有摄取:** 在 Stratum 升级前，AII 继续用现有降级路径摄取，不等待。
