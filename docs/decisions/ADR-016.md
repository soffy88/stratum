# ADR-016 — Stratum 本地优先部署策略

**状态**: 已确认 (决策时间不详) → 拓扑扩展 (见 ADR-021, 2026-05-20)
**决策者**: Wiki (Stratum chief advisor)

---

## 决定

Stratum 采用本地优先 (local-first) 部署策略：

- 所有核心服务 (数据库、向量索引、全文索引、消息队列) 部署在用户本地机器
- 不依赖外部 SaaS 托管数据库或向量数据库云服务
- substrate 数据存储在本地 + 用户自有网盘 (GDrive)，不进入第三方云存储
- 外挂能力 (AI 推理) 优先本地部署，云 API 仅作降级备选

## 理由

- 隐私: 个人知识库内容不离开用户控制的基础设施
- 可靠性: 不依赖外网服务可用性
- 成本: 无 SaaS 订阅费用

## 拓扑扩展标注

**2026-05-20 — ADR-021 扩展**: 本地部署从"单机"扩展为"三机本地网络"拓扑：
