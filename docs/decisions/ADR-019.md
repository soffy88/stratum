# ADR-019 — Stratum 外挂能力架构

**状态**: 已确认 (决策时间不详) → **部分修订 (ADR-021, 2026-05-20)**
**决策者**: Wiki (Stratum chief advisor)

---

## 决定

Stratum 外挂能力 (AI 推理服务: whisper / TTS / SD / searxng / ollama 等) 采用以下集成架构：

- 各外挂独立 Docker container，独立故障域
- 外挂与 Stratum 主体通过 Docker network 通信 (原始决定: 同机 Docker network)
- 各外挂经理人独立，Stratum advisor 只规范接入接口 (不管外挂内部实现)
- 本地优先: 不依赖云 API
- MCP 协议优先通信: 外挂经理人通过 MCP 向 Stratum 注册工具

## 理由

- 解耦: 各外挂独立升级、独立重启，不影响 Stratum 主体
- 职责分离: 各经理人只需暴露规范接口
