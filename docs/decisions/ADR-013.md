# ADR-013: SPEC 文档纪律


**日期**: 对话末期 (Wiki 多次强调)
**状态**: ✅ 已内化

### 决定
SPEC v0.5 写作纪律:

1. **对工程实施没用的不写**
   - 删: 变更摘要 / 对照表 / 产品营销定位 / 长期主义等空话
2. **不写"反向定义不做什么"**
   - 这是产品决策, 不是工程约束
3. **每条原则必须能翻译成工程约束**
   - "数据归用户" → "服务器不持有 substrate 原始 bytes"

### 应用
- SPEC v0.3 → v0.5 应用此纪律
- v0.5 1214 行 (比预估 2000-2500 行少, 因为剔除冗余)
