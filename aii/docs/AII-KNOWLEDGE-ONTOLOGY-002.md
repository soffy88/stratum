## 〇、本体论的诚实前提

在定义任何东西之前，先承认一个事实：**知识的分类在哲学上没有定论，不存在一份被公认的"知识类型总表"(master list)。** 知识是纯哲学议题，争论跨越数个世纪。

因此本文档**不宣称发现了知识的"真本体"**。它做的是一件更克制、也更可靠的事：从知识表示与认识论中提取经得起检验的结构，构造一套**对 AII 工程上够用、且不违背学理**的知识单元定义。这是一个"足够好的实用本体"，不是终极真理。这一克制本身就是知识界的基本诚实，也是 AII"可置信"的起点——不把"看起来完备"冒充"真的完备"。

### 〇.1 本文的定位：元本体（依 AII-DOMAIN-ONTOLOGY-SPEC-001 §1.3）

上面那段克制**不等于"随便定一套够用就行"**。补足定位如下，免得被读成工程派的"本体=任意的概念化规格"（那条路必然增殖出千千万万互不相容的本体，是死局）：

**本文是 AII 的元本体，对应 OBO 生态里 BFO 的角色**——极简、领域中立的元语法，供各领域本体挂靠。与之相对，**领域本体不在这里定义，而是从书中被发现的**（B仓 canonical 概念层 + 判别维度 + 第①层有向关系，对应 GO 等领域参考本体）。

**"人定"但不任意：它是关于"知识的真实结构"的一个假设，因而可被证据推翻并修正。** 这不是空头承诺，已经发生过至少四次：

| 修正 | 证据 |
|---|---|
| 本性术语 nature → invariant 全系统统一 | commit `c1d35ad`（2026-06-25） |

所以：**本体是被发现的，不是被规定的**；元本体这一层由人给出假设，但以实在为基准、随证据修正，不是不可动的教条（DOMAIN-ONTOLOGY-SPEC-001 R4）。旧表述"本体 = 人预先规定的框架"已作废。

## 一、什么是知识单元(KU)：本体定义

### 1.1 知识表示的本源

知识表示领域有一个被反复引用的根本定义（Davis, Shrobe & Szolovits）：

> 一个知识表示首先是一个 **surrogate（替代物）**——是被表示的事物本身的替身，使一个主体可以"通过思考而非行动"来推断后果；它同时是**一组本体承诺(ontological commitments)**，即对"世界里应当假设存在哪些东西"的回答。

这给了 KU 最底层的定位：**KU 不是文本，是被表示之物的替身。** 一段关于"损失厌恶"的文字不是 KU；"损失厌恶"这个被该文字所指的、可以拿来推理的知识对象，才是 KU。文本是载体，KU 是载体所替代的那个东西。

### 1.2 原子知识单元的收敛定义

多个相互独立的来源（知识元理论、原子知识单元 AKU、知识库工程实践）对"最小知识单元"给出了高度一致的刻画。综合如下：

> **一个 KU 是一个可被命名、标引、存储、检索、独立操作的最小自足知识包；它恰好对应单一一个连贯的事项；移除它的任一必要组成部分，都会降低基于它正确推理或行动的能力。**

由此提炼 KU 的**四个本体属性**：

1. **原子性(Atomicity)**：恰好承载**单一一个连贯事项**(addresses exactly one coherent subject)。再拆则不完整，合并则裹入多个事项。这里的判据是"单一事项"，比"一句话"更精确——一个事项可能需要几句话才说清，但仍是一个 KU。

2. **自足性(Self-containedness)**：脱离原始上下文仍可被理解和陈述。一个只有放回原文段落才看得懂的片段，不是合格 KU。

3. **可陈述性(Assertability)**：是一个**可被断言的知识**（一个命题、一个概念、一项方法、一种解释、一个立场），而非一个"问题"或"任务"。
   > **认识论理由:** 知识的传统对象是**命题**——可以判断真假的陈述。一个"问题"(如"为什么会通货膨胀？")本身不是命题、无真值，因此不是知识单元。它的**答案**才可能是。这条排除有坚实的认识论根据，不是工程约定。
   > （注：AII 系统中若存在"待解问题"类记录，应理解为**研究方向标记**，归属于元层，而非 KU 本体的一员。）

4. **可关联性(Connectivity)**：能与其他 KU 建立关系，纳入知识网络。一个连不上任何东西的孤岛，本体上是可疑的——见第五节。

### 1.3 一个关键澄清：可评测性不是本体属性

"能不能测量掌握度"**不是** KU 的本体属性。它只是某一类知识（可完全用语言陈述的那类，见 2.1）的表象后果。把"可评测"写进 KU 的定义，会把大量真知识——处世智慧、鉴赏方法、哲学立场、机制性理解——错误地排除在外。一个外脑若只收"可考的"，它就不是外脑，是题库。

## 二、知识的类型：认识论的三分 + 一个相对性维度

知识不是单一类型的东西。**这是 AII 该如何存储知识的根本依据**——不同类型的知识，本体结构不同，存法不同。

本节的理论根**不来自任何教育测评框架**，而来自**认识论哲学**对知识种类的经典区分。

### 2.1 三分法：know-what / know-how / know-why

认识论中一个跨越 Ryle 到当代认知科学的核心区分：

| 类型 | 哲学名 | 本质问题 | 能否完全言传 |
|---|---|---|---|
| **陈述性知识** | know-that（命题性知识） | "X 是什么？关于 X 什么为真？" | 能，可完全用命题表达 |
| **程序性知识** | know-how | "怎么做？" | **不能完全言传**，靠练习获得 |
| **释因性知识** | know-why（理解 / understanding） | "为什么？机制如何运作？" | 部分能，但不等同于罗列事实 |

**这三类不可相互还原**，这有认知科学的硬证据支撑：

> 陈述性系统与程序性系统在大脑中是**分离的**(Stanford Encyclopedia)。Ryle 的反智主义论证——know-how 不可还原为 know-that——正建立在此之上。

**这一条对 AII 有两个直接后果：**

- **程序性知识(know-how)天然"难以言传、靠练习获得"**，所以它"不能用对错考、要靠引导式练习"——**不是它不算 KU，而是它是程序性 KU，学习/呈现方式天然不同**。这解决了"鉴赏、表达、处世算不算知识"的长期纠结：它们是 know-how。

- **释因性知识(know-why)是独立的一类，极易被遗漏。** 一个只抽出"损失厌恶是什么"(know-what)、却没抽出"它为何发生、机制如何"(know-why)的知识库，是**浅的**。
  > **这很可能是"理解深度不够"的本体根源**：当抽取只盯着命题(know-what)，know-why 被系统性丢弃，知识网络就只有"是什么"的骨架，没有"为什么"的血肉。**深度 = know-why 的在场。**

### 2.2 陈述性知识的内部细分

陈述性知识(know-what)在知识表示中通常进一步分为三种形态，用于决定它们如何被结构化：

- **事实性(Factual)**：特定的、可考证的记录。"赤壁之战发生于公元 208 年""水的沸点是 100°C"。抽象度低，迁移性弱。
- **概念性(Conceptual)**：事物本质与意义的抽象。包含分类(圆锥曲线分为椭圆/双曲线/抛物线)、原理与规律(勾股定理、供求规律、损失厌恶)、理论与模型(进化论、前景理论)。
- **关系性(Relational)**：知识表示中常被单列的一类——知识本身就是一个**关系**(A 导致 B、A 是 B 的特例)。在 AII 里它既可以是 KU，也常常体现为 KU 之间的边(见第六节)。

> **同一个对象会沿不同类型展开为多个 KU。** 以"勾股定理"为例：
> - "勾股定理：a²+b²=c²" → 陈述性·概念性（命题）
> - "用勾股定理求边长的步骤" → 程序性（know-how）
> - "勾股定理为何成立 / 它刻画了直角三角形的什么本质" → 释因性（know-why）
>
> **注意这与"认知层级"的区别。** 知识表示的一个重要结论是：**知识类型(它是什么知识)与对它的认知操作(记住它/运用它/分析它)是两个正交的维度。** "勾股定理"的类型是恒定的(概念性/命题)；"记住"还是"运用"它，是认知操作的不同，**不改变它的类型**。
> 所以上面的拆分**不是按"认知层级"拆**(那样会把认知操作混进类型)，而是因为"是什么(命题)"、"怎么用(程序)"、"为什么(解释)"**本就是三种不可还原的知识**。这是本体的拆分，不是难度的拆分。

### 2.3 相对性维度：立场性知识

认识论中还有一个无法回避的事实：**并非所有命题都有确定的真值。**

在思想类领域（哲学流派、社科理论、经济学派），存在大量**有争议、无定论、靠论证强弱而非真假来衡量**的知识："存在先于本质"(萨特)、凯恩斯主义 vs 奥地利学派、精神分析 vs 行为主义。这里没有"对的那一个"。

对这类知识，把它当作客观事实呈现是**认识论错误**。它必须被表示为**持有者相对的命题**——

> **立场性知识 = 一个相对于其持有者(stance holder)的命题，其地位由论证强度而非真值决定。**

这不是"第五种并列的知识类型"，而是**陈述性知识的一种特殊样态**：当一个命题缺乏确定真值、只在某个立场内部成立时，它就进入立场性。表示它的**充要条件**是同时记录：

- **立场持有者(stance holder)**：这是谁的主张。"潜意识冰山理论 — 持有者：弗洛伊德/精神分析学派"。
- **对立立场**：与之竞争的其他立场。

**绝不能脱离持有者把立场陈述为事实。** 这是个人外脑区别于教科书的核心诚实之一。

## 三、知识的认识论地位：可信度与时效

一个 KU 不只"是什么类型"，还有它的**认识论地位**——它有多可信、是否仍然成立。这是"可置信外脑"的命门，也是上一版完全遗漏的本体维度。

### 3.1 可信度(epistemic standing)是 KU 的本体属性

认识论的基本要求：**知识需要认识论根据(epistemic standing)——你必须有理由或证据，而不只是恰好猜对。** 一个无根据的断言，即使为真，也不构成知识(JTB 传统)。

因此每个 KU 必须携带它的**可信度**，且可信度有**来源**：

- **默认状态：未经证实(unverified)。** 任何新抽取的 KU，在被核验之前，只是一条线索，不是确证的知识。这是诚实的默认——不预设它为真。
- **可信度通过核验机制提升。** 不同类型的知识，核验方式不同：数学命题可经形式化系统(如证明检查)确证；事实性知识靠权威来源交叉；立场性知识则**永远不会变成"确证为真"**，它的"可信"指向的是论证强度，不是真值。

> **关键:** 可信度**不是 LLM 随手标注的**，而是要么默认未证、要么经机制确证而来。"它说它对"与"它真的对"是两回事——这是 AII 全部价值的支点。

### 3.2 时效与可修订性：知识是活的

认识论中一个常被工程忽略、却至关重要的洞见：**知识不是死的、入库即终结的。理解是临时的，随新证据被修订。**

> 同样的内容，随认知推进，知道(knowing)会变化，理解通过反馈和经验被修订。

因此 KU 的本体必须包含**时效维度**：

- 一个 KU 有其**成立的时间范围**——它可以被更新的知识**取代(supersede)**或**推翻**。
- 当新知识与旧 KU 冲突，正确的做法不是删除旧的，而是标记其**失效/被取代**，保留演化痕迹。

**这是个人外脑区别于静态笔记的根本之一。** 一个不会自我修订的知识库是标本馆；一个活的知识库会随新知识进来重新调整。可修订性是 KU 本体的内在属性，不是附加功能。

## 四、KU Schema（本体结构）

```yaml
KU:
  id:               唯一标识
  name:             知识单元名称（命题 / 概念 / 方法 / 解释 / 立场）
  source:           出处（书/论文 + 位置）

  # ── 知识类型（认识论三分 + 形态细分）──
  knowledge_kind:   declarative | procedural | rationale   # know-what / how / why
  declarative_form: factual | conceptual | relational        # 仅 declarative 时填
  is_positional:    bool                                      # 是否为立场性（持有者相对）

  # ── 核心内容（按 knowledge_kind 填不同字段）──
  content:
    # declarative-factual:   5W（时间/地点/人物/事件/意义）
    # declarative-conceptual: 定义/内涵/外延/原理表述/适用条件
    # procedural (know-how):  步骤/方法要点/适用条件/典型示例
    # rationale (know-why): 机制/因果链/"之所以如此"的解释
    # positional:            主张/论证/对立立场

  intuition:        直观理解 / 为什么重要 / 反直觉点（产生"回响"的关键）
  insight:          深层洞察（释因性与思想类知识尤其重要）
  example:          应用中的实例

  # ── 立场性附加（is_positional=true 时必填）──
  stance_holder:    主张者 / 学派
  opposing_stance:  对立立场

  # ── 认识论地位 ──
  epistemic_standing:                                          # 见 §3.1
    grade:          unverified（默认）| verified | refuted | ...
    grounded_by:    可信度来源（默认未证 / 经某机制确证 / 立场内论证）
  temporal:                                                    # 见 §3.2
    valid_from:     成立起始
    valid_until:    失效时点（可空）
    superseded_by:  被哪个 KU 取代（可空）

  # ── 知识网络关系 ──
  relations:
    - type:  上下位 | 因果 | 对比 | 对立立场 | 前置依赖 | 支撑 | 跨源关联
      target: 目标 KU id
```

## 五、KU 准入：什么不配做一个 KU

> 本节是质量命门。§1 回答"什么是 KU"，本节回答"**什么不配做 KU**"——后者往往比前者更能决定整个知识网络的价值。

### 5.1 第一性原理：连接 > 收集(Connect, not Collect)

知识网络的价值不在于"存了多少"，而在于"能连接多少"。一个连不上任何东西的孤立条目，本质上没有知识价值——它违背了 §1.2 的可关联性。**判断一个候选配不配做 KU，第一性原理是看它能否进入网络，而非能否被存下。**

### 5.2 要警惕的失败模式：收集的幻觉

人倾向于囤积信息而不对它做任何有用的事：收藏夹塞满从未打开的链接，笔记堆满未经咀嚼的摘抄。**除了制造"读过"的幻觉，囤积本身不带来任何帮助。** 把一切都当 KU，正是落入此陷阱。

### 5.3 判据一：信息 vs 知识（能否用来推理）

认识论对"信息"与"知识"有明确分野（DIKW 传统）：**data 是未处理的符号；information 是被组织、被赋予相关性的 data；knowledge 产生于运用 information 去理解模式与原理。** 关键分野是：

> 一段内容，只有当它经过阐述、能被用来推理时，才从"信息"升格为"知识"。

→ 纯数据点、一次性背景陈述、未消化的摘抄 = 信息，不是 KU。**一份完整的文档(如一篇报告)整体是 information(被组织的内容)；它所包含的 knowledge，是从中提炼出的"为什么如此、机制如何"的可推理判断。**

### 5.4 判据二：原材料 vs 成品（拆解，而非堆放）

> KU 库装的是**原材料**，不是**成品**。

成型的论述要被肢解成可重新组合的原材料。就像搭积木——库里放的是一块块积木，不是搭好的城堡。

→ 整段论述、完整故事、案例全文 = 成品。它们要被**拆解**成背后的概念/解释/原理（原材料），那些才是 KU。

### 5.5 判据三：未来可被需要（最可操作的黄金标准）

面对一个候选，不要问"我该把它存到哪个分类"，而要问：

> **"即使我忘了它，未来在什么具体情境下，我会希望这条知识自己冒出来？"**

- 答得出具体情境 → 它是值得记的 KU（在未来有"被需要"的潜力）。
- 答不出 → 它是噪音/背景，不配单独成为 KU。

### 5.6 论据 ≠ KU

书中大量的故事、实验、案例是**论据**，不是知识单元；它们支撑的那个观点/概念/解释才是 KU。

> 一堆赌博实验（论据）→ 提炼出"损失厌恶"（KU）。实验本身降级为该 KU 的 `content.example` 或一条"支撑"关系，不单独成 KU。

**但案例的知识地位是动态的**：被反复引用、获得独立学科地位的标志性事件（如米尔格拉姆服从实验、巴甫洛夫的狗），可**升格为事实性 KU**。判断标准：它是"一次性论据"，还是"被反复引用、值得独立讨论的典范"。

### 5.7 关于量：没有预设标准

> **一本书有多少 KU，取决于它承载了多少通过上述准入的真知识——这是结果，不是目标。**

- 一本知识密集的扎实著作（如一本严谨的微观经济学教材），概念、原理、方法、解释密集，**且每个对象还沿 know-what/how/why 展开为多个 KU**——它产出几千甚至上万个 KU，是对的。
- 一本翻来覆去只有几个观点、其余皆是故事与水分的书，产出几个 KU、甚至接近零，也是对的。

**不预设"一本书该有几十个 / 几百个 KU"。** 任何这样的预设都违背"量由知识密度决定"这一本体事实。准入闸门保证的是**质**（每个 KU 都是真知识、能连接、已拆为原材料），不是**量**。压缩比、数量这些指标只能用来**警示异常**（如一本厚书只抽出几条，提示抽取失败），**不能用来设定目标**。

---

## 六、知识网络：KU 不是列表，是一张图

KU 之间的关系往往需要从连续的论述流中**重建**。关系类型：

| 关系 | 含义 | 例子 |
|---|---|---|
| 上下位 | 概念的包含/从属 | 前景理论 ⊃ 损失厌恶 |
| 因果 | 一个导致另一个 | 锚定效应 → 调整不足 → 估计偏差 |
| 对比 | 并列对照 | 系统1（快/直觉） ↔ 系统2（慢/理性） |
| **对立立场** | 互相竞争的立场 | 精神分析 ⟂ 行为主义 |
| 前置依赖 | 理解顺序 | 函数概念 → 导数 |
| 支撑 | 论据支撑论点 | 实验 → 理论（实验是论据，非独立 KU） |
| **跨源关联** | 不同来源讲同一底层知识 | 论语"己所不欲" ↔ 非暴力沟通"换位思考" ↔ 同理心 |

> **跨源关联是 AII 的终极价值。** 不同书讲同一个底层知识时，在网络里汇聚。
>
> **本体决策：同一知识的跨源出现，是"合并为一个 KU + 多来源"，而非"多个 KU + 关联边"。** 因为 §1.1 说 KU 是被表示之物的替身——同一个被表示之物只应有一个替身，多个来源是它的多个出处与多种表述，挂在同一个 KU 上。这与"跨源关联"关系不冲突：关联用于**不同但相关**的知识之间；合并用于**同一**知识的多次出现。

## 七、从一本书/一个领域抽取 KU 的方法论

### 第一步：识别知识形态（决定抽法）

| 形态 | 典型 | 抽取重点 |
|---|---|---|
| 概念体系型 | 微观经济学、普通心理学 | 抽概念，理清概念网络；**对每个概念追问 know-why** |
| 论点论证型 | 思想类专著 | 抽命题（论点）+ 解释（why）+ 论证 |
| 方法实践型 | 方法论、处世类 | 抽原则性命题与方法（程序性 know-how） |
| 思想立场型 | 哲学、流派 | 抽立场 + 持有者 + 对立观点 |
| 叙事型 | 小说、传记 | 抽洞察/主题（不强行抽 KU） |

### 第二步：定位知识承载点（覆盖全书，不挑重点）

**遍历全书每一个章节、每一个知识承载点**——抓住承载知识的位置：核心论点、反复出现的术语、"本质是…""关键在于…""之所以…是因为…"等信号词后的句子、案例后的"这说明了…"。
> **不是"挑重要的抽"，是"覆盖每一处，逐个判断"。** 挑重点会系统性漏掉整章。覆盖全书 + 准入闸门，才能既不漏、又不滥。

### 第三步：过准入闸门（见第五节）

逐个候选过准入测试（连接>收集、信息vs知识、原材料vs成品、未来可被需要）。**大部分内容在这一步被过滤或降级**：通过 → 成为 KU；不通过但支撑某 KU → 降级为该 KU 的 example/支撑关系；不通过也不支撑 → 丢弃。

### 第四步：套本体标准提炼 + 归类

每个候选用四属性（原子/自足/可陈述/可关联）检验；判定 knowledge_kind（know-what/how/why）；若是立场性，标 stance_holder/opposing_stance；标 epistemic_standing（默认 unverified）。**对每个概念，主动追问是否存在对应的 know-why（解释）尚未抽出。**

### 第五步：重建关系 + 跨源关联

抽完一本书应能画出该书的"知识地图"；单本 KU 进网络后与其他来源的 KU 连接或合并。

## 八、设计要点小结

1. **KU 是被表示之物的替身（surrogate），不是文本。** 本体属性 = 原子性（单一事项）+ 自足性 + 可陈述性 + 可关联性，与"能否评测"无关。

2. **知识分三类，不可相互还原**（认识论 + 认知科学）：陈述性（know-what）、程序性（know-how）、**释因性（know-why）**。陈述性再细分事实/概念/关系。**know-why 的在场 = 深度的来源**——这是上一版遗漏、却最关键的一类。

3. **立场性是陈述性的特殊样态**：当命题无确定真值、只在某立场内成立，必须记录持有者与对立立场，绝不当作事实。

4. **可信度与时效是 KU 的本体属性**：默认未经证实，经核验机制提升；知识是活的，可被新知识取代/推翻，保留演化痕迹。这是"可置信外脑"的命门。

5. **准入闸门是质量命门**：连接>收集、信息vs知识、原材料vs成品、未来可被需要。**不是什么都配做 KU。**

6. **量由知识密度决定，不预设**：密集的好书几千上万个 KU 是对的，水书几个也是对的。数量只用于警示异常，不用于设定目标。

7. **同一知识跨源 = 合并为一个 KU（多来源）**；不同但相关的知识 = 关联边。**跨源汇聚是 AII 区别于普通笔记的终极价值。**

8. **诚实前提**：知识分类在哲学上无定论，本套类型是"工程上够用且不违背学理"的实用本体，不宣称是终极真理。这一克制本身就是 AII"可置信"的一部分——不让"看起来完备"冒充"真的完备"。
